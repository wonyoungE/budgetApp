# AGENTS.md

## 프로젝트 설명
이 저장소는 `CSV` 기반 가계부 `Python CLI` 앱 프로젝트다.

목표는 CSV 파일로 거래 내역을 읽고, 거래를 추가하고, 잔액과 카테고리별 요약을 CLI 환경에서 다루는 것이다.

## 코딩 규칙
- 모든 함수와 공개 인터페이스에는 타입 힌트를 반드시 작성한다.
- 함수 하나의 길이는 최대 50줄을 넘기지 않는다.
- 함수는 한 가지 책임만 갖도록 쪼갠다.

## TDD 규칙
- 구현 전에 반드시 테스트를 먼저 작성한다.
- 실패하는 테스트를 확인한 뒤에만 구현을 시작한다.
- 기능이 완성되면 리팩터링 전에 테스트가 통과하는지 확인한다.

## 품질 규칙
- 순환복잡도는 10 이하로 유지한다.
- 복잡도가 높아지면 로직을 분해하고 조건문을 단순화한다.

## 품질 검수 규칙
- 커밋 전에 반드시 `qa_engineer` 서브에이전트로 품질 검수를 수행한다.
- 품질 검수에서는 테스트 누락, 복잡도 초과, 타입 힌트 누락, 함수 길이 초과를 확인한다.

## 테스트 실행 명령어
- `pytest`
- `radon cc`

## 커밋 규칙
- 하나의 기능이 개발되면 즉시 커밋하고 푸시한다.
- 기능 단위로 커밋 범위를 나눈다.
- 커밋 전에 테스트와 품질 검수를 완료한다.
